User Guide

Overview

Config Builder creates Cradlepoint .bin configuration files from templates with per-site variable substitution. No internet or NCM API access required.

Workflow

  1. Create a Config Template — Go to Config Templates and click Add Config Template. Give it a name, upload or paste a Base config (connectivity on install), then click Next and upload or paste a Full config (applied by remote admin). Save the template.
  2. Add variables — In your JSON configs, replace site-specific values with {{Variable Name}} placeholders. Example:
    "ip_address": "{{Vlan2 IP Address}}"
  3. Build a site config — Go to the Build tab. Select your template. Fill in all the fields (site info + template variables). The form is auto-generated from your template placeholders.
  4. Save the site — Enter a filename (default: sites.csv) and click Save Site. This stores the site data for reuse later.
  5. Download configs — Click Download Base Config or Download Full Config to get the .bin files with all variables replaced. Files are named like:
    RDL999222 - E3000 - Full Config.bin
  6. Load a saved site — Switch to "Load Saved Site" to pick a previously saved site, edit values if needed, and re-download configs.

Variable Types

By default variables are strings. To enforce a type, use {{name|type}}:

  • string — any text
  • integer — whole number (e.g. 42)
  • float — decimal number (e.g. 3.14)
  • boolean — true or false
  • ipv4 — IPv4 address (e.g. 192.168.1.1)
  • ipv6 — IPv6 address
  • cidr — CIDR notation (e.g. 10.0.0.0/24)
  • mac — MAC address (e.g. AA:BB:CC:DD:EE:FF)

Tips

  • You can upload .bin files exported from NCM — they are automatically decompressed to JSON.
  • If you download without saving, you'll be prompted to save first.
  • Manage saved site CSVs from the Saved Sites tab.
  • Router models can be extended by editing router_models.csv.
